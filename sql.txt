INFORMATION_SCHEMA.TABLES
INFORMATION_SCHEMA.TABLES is a system view provided by most relational databases 
Purpose: Give you a consistent way to query database metadata (tables, columns, constraints, etc.) without relying on vendor-specific system tables.

INFORMATION_SCHEMA.TABLES contain :

TABLE_CATALOG → Database name
TABLE_SCHEMA → Schema name (e.g., dbo, prc)
TABLE_NAME → Table name
TABLE_TYPE → 'BASE TABLE' (real table) or 'VIEW'

backup collection/table
we can backup collection in mongo by $merge pipe line which gets applied on source collection 
e.g
  db[sourceCollection].aggregate(    {
      $merge: {
        into: backupCollectionName,
        whenMatched: "keepExisting", // keep first snapshot of the day
        whenNotMatched: "insert"
      }
    });
